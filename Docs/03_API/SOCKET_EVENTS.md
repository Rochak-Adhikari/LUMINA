# Lumina V2 Socket.IO API Specification

The frontend React client communicates with the backend via bidirectional WebSockets on port `8000`.

---

## 1. Emitters (Sent from React Client to Backend)

### Connection & Session

#### `start_audio`
- **Description**: Registers client socket, configures microphone devices, and spawns the `AudioLoop` thread.
- **Payload**:
  ```json
  {"mic_device_index": 1, "speaker_device_index": 2}
  ```

#### `stop_audio`
- **Description**: Stops the live `AudioLoop` and releases pyAudio devices.

---

## 2. Listeners (Received by Client from Backend)

### Real-Time Streams

#### `audio_data`
- **Description**: Relays raw PCM audio chunks generated by the speaker loop. Feeds the visualizer canvas.
- **Payload**: Binary buffer.

#### `transcription`
- **Description**: Streams user speech-to-text words or Lumina reply lines.
- **Payload**:
  ```json
  {"text": "Mero color settings kacha?", "sender": "User"}  // or "Lumina"
  ```

#### `browser_frame`
- **Description**: Delays headless browser tab screenshots during CDP browser control loops.
- **Payload**: Base64 image payload.

#### `cad_data`
- **Description**: Streams OpenSCAD STL geometry files to render in the client canvas.
- **Payload**: Base64 compiled model.

---

## 3. Config & Settings Events

### Emitters (Client → Backend)

#### `get_settings`
- **Description**: Requests the complete configurations dictionary.

#### `update_settings`
- **Description**: Persists updated config fields.
- **Payload**:
  ```json
  {"persona_mode": "playful_mischievous_best_friend", "vad_min_speech_ms": 300}
  ```

### Listeners (Backend → Client)

#### `settings`
- **Description**: Dispatches config states, pairing QR codes, and allowed tool lists.
- **Payload**:
  ```json
  {
    "persona_mode": "playful_mischievous_best_friend",
    "tool_permissions": {"generate_cad": false, "browser_control": true},
    "remote_pairing": {"qr_url": "http://192.168.1.50:8000/pair/123"}
  }
  ```

---

## 4. Operational Control

### Emitters (Client → Backend)

#### `kill_browser_tools`
- **Description**: Immediately terminates all active Playwright/CDP sessions.

#### `reminder_alarm_dismissed`
- **Description**: Acknowledges and closes an active calendar alarm.
- **Payload**:
  ```json
  {"event_id": 5}
  ```

#### `tool_confirmation_response`
- **Description**: Approves or rejects a gated tool function call.
- **Payload**:
  ```json
  {"id": "confirmation-uuid-token", "confirmed": true}
  ```

### Listeners (Backend → Client)

#### `tool_confirmation_request`
- **Description**: Displays the confirmation modal for gated operations.
- **Payload**:
  ```json
  {
    "id": "confirmation-uuid-token",
    "tool": "local_browser_control",
    "args": {"action": "click_text", "text": "Pay Now"}
  }
  ```

#### `reminder_alarm`
- **Description**: Displays the fullscreen alert screen for an active event reminder.
- **Payload**:
  ```json
  {"id": 5, "title": "Team Sync", "dt": "2026-07-16T17:00", "notes": "Meeting link..."}
  ```
